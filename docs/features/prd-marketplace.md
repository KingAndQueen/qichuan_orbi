## 1. 一句话摘要
为企业提供统一的工作流发现与管理平台：既支持原生的[TERM-IA-015]工作流方案，也支持通过标准化适配器接入第三方生态智能体（兼容模式）；提供从购买、授权分发到回收、续费的统一全生命周期管理，确保企业对内外部AI能力的引入均可控、成本透明、结果可验证。

## 2. 目标与指标
### 2.1 产品目标
- G0-1（安全基线）：透明授权与边界管控：订阅前，管理员能清楚看到“这项能力会访问哪些数据、能做哪些操作”，并能识别高风险权限；订阅后，任何超出[TERM-IA-024]权限声明清单范围的操作默认被拦截并留下可追溯记录。（规则：R7、R14）
- G0-2（生态基线）：连接标准开放性与生态兼容：建立或采用行业主流的标准化连接协议，确保系统能以低成本、标准化的方式接入外部工具（如 SaaS API）及外部智能体（如 Coze/扣子、阿里百炼、飞书 Agent）；避免为单一来源定制私有非标连接器。（规则：R12、R14）
- G0-3（商业基线）：确立[TERM-MP-004]数字化雇佣“价值体系”：构建支持多种薪酬模式的混合计费引擎，不仅支持基础的[TERM-MP-006]按岗雇佣，更支持面向未来的[TERM-MP-005]按结果付费，全面适配数字劳动力带来的商业模式变革。（规则：R13）
- G1：提供按角色分级可视的能力目录与分发入口：管理员视图可按业务分类浏览全量可订阅能力，并在卡片上看到做决策所需的关键信息（包含成本标签（按岗雇佣单价/按结果付费单价））；员工视图“我的应用”仅展示已被授权/分配的能力，确保员工不被无权限内容干扰、也不会误用越权能力。（规则：R1、R2）
- G2：管理员能在市场一处完成 B2B 资产闭环管理：订阅/购买→授权分发→回收→增购/续费→订单管理；并提供清晰的订阅管理入口（如“我的订阅”）便于增购/续费与订单查询。管理员可随时看清“[TERM-IA-019]可用额度”与“[TERM-IA-020]已分配额度”的归属（分配给哪个对象/团队）。（规则：R3、R4、R6、R8）
- G3：已安装能力的运行版本保持稳定，避免“同一任务前后表现不一致”；当有新版本可用时，仅提示可升级，升级必须由管理员手动触发并确认变更。（规则：R5）
- G4：能力运行边界清晰且可控：只在授权范围内访问与执行；任何越界尝试都会被系统阻止并对管理员可见。（规则：R7）
- G5：支持在同一预定义[TERM-IA-005]工作流中自由选择或移除多个[TERM-IA-007]智能员工，形成不同的工作流方案与价格：系统能明确每一步“需要什么输入、会产出什么结果”，并自动把上一步结果交给下一步；当用户调整智能员工组合时，系统实时校验方案是否可运行，并提示对交付范围、质量与时长的影响。（规则：R9、R10、R11）

### 2.2 成功标准

- 成功标准 S1：管理员可闭环完成“订阅/购买→授权分发→回收→增购/续费→订单管理”，并可在订阅管理入口完成增购/续费与订单查询。（规则：R3、R4、R6）
- 成功标准 S2：管理员可在市场页按分类浏览全量可订阅能力，并在卡片上看到关键决策要素（包含成本标签（按岗雇佣单价/按结果付费单价））；员工仅在“我的应用”看到已被授权/分配的能力，并可直接开始使用（例如进入对应工作入口）。（规则：R1、R2）
- 成功标准 S3：订阅前必须清晰展示权限范围；订阅后任何超出已授权范围的操作都会被系统拦截并可追溯。（规则：R7）
- 成功标准 S4：管理员可在工作流详情中自由调整智能员工组合；系统实时给出可运行性校验结果与降级影响提示，并在组合可运行时生成对应价格与可订阅方案。（规则：R10、R11）
- 成功标准 S5：已安装的工作流方案在使用期间保持版本稳定；市场侧更新不会自动改变企业当前正在使用的版本；管理员确认并手动触发升级后，新版本才会生效。（规则：R5）
- 成功标准 S6：工作流方案支持多智能员工分步协作：系统能在运行时自动完成步骤交接（上一步输出自动成为下一步输入），让同类任务可以顺畅复用并稳定交付。（规则：R9）

### 2.3 指标定义与统计口径（Metrics，统计口径不强制事件名）

#### 北极星指标（North Star Metric）

- 指标名：关键闭环完成率
  - 对应成功标准：S1、S2
  - 一句话定义：统计周期内，发生“订阅/购买并完成授权分发”的工作流方案中，至少有 1 次员工从“我的应用”开始使用的比例。
  - 存在理由：市场的核心价值不是“被浏览”，而是把可付费能力变成“可被授权、可被使用”的闭环；用于评估资产闭环与分级可视是否真正产生业务结果（对应目标：G1、G2）。
  - 指标类型：ratio
  - 分析对象：workflow_plan（工作流方案）
  - 时间粒度：day
  - 时间窗：统计周期（如自然日/自然周/自然月；按报表选择）
  - 过滤条件：排除内部测试账号、自动化回放、异常数据；同一组织多次购买按订单记录计入（TBD）。
  - 维度（可选）：工作流 / 方案价格区间 / 行业 / 套餐 / 客户端版本（TBD）
  - 数据来源：市场订阅/购买、授权分发、员工开始使用相关行为事件/表（TBD；与 `docs/data/metrics.md#marketplace` 对齐）。
  - 计算口径：
    - （ratio 类型才需要）
      | 组成项 | 定义 | 去重口径 | 过滤条件 |
      |---|---|---|---|
      | 分子 | 完成“订阅/购买 + 授权分发”且统计周期内至少发生 1 次员工开始使用的工作流方案数 | 按 workflow_plan_id 去重 | 同上 |
      | 分母 | 完成“订阅/购买 + 授权分发”的工作流方案数 | 按 workflow_plan_id 去重 | 同上 |
      - 公式：分子 / 分母

## 3.2 规则表
| 规则编号 | 规则名 | 规则描述 | 关联目标 |
| --- | --- | --- | --- |
| R1 | 员工端仅展示授权应用 | Employee 侧仅展示已被授权/分配的能力（我的应用），并提供一键开始使用的入口（例如进入对应工作入口）。 | G1 |
| R2 | 管理员端全量市场浏览 | Admin 侧必须提供全量市场浏览能力：至少包含分类导航 + 卡片流，并在卡片上展示关键决策要素（名称、简介、供应商、成本标签（按岗雇佣单价/按结果付费单价）），以支持管理员快速筛选与订阅决策。 | G1 |
| R3 | 工作流方案授权分发可回溯 | 订阅/购买后必须支持将[TERM-IA-015]工作流方案对应的[TERM-IA-016]订阅实例进行[TERM-IA-017]授权分发到指定对象，并保证分发结果可被确认与回溯。 | G2 |
| R4 | 订阅管理与订单查询 | 必须提供订阅管理入口（如“我的订阅”），管理员可查看已订阅/已购买的工作流方案，并完成增购/续费与订单查询。 | G2 |
| R5 | 版本快照锁定与手动升级 | 安装即锁定版本快照；更新采用被动策略，升级必须由管理员手动触发。 | G3 |
| R6 | 回收与额度回流 | 必须支持手动回收；回收生效后对应额度立即回流为可再次分发的可用额度，并可选支持 30 天未活跃自动回收。 | G2 |
| R7 | 权限声明与默认拦截记录 | 订阅前必须展示权限声明清单；订阅后对未在清单中授权的操作默认拦截，并产生对应拦截记录。 | G0-1、G4 |
| R8 | 试用策略与到期回收 | 必须支持试用额度的固定策略与到期自动回收。 | G2 |
| R9 | 多员工分步协作、信息需求与声明 | 必须支持多智能员工分步协作，并提供标准化的声明供系统编排：至少包含 [TERM-WS-015] 输入输出声明；当步骤依赖企业知识时，还必须支持声明 [TERM-WS-018] 信息需求，并允许引用 [TERM-WS-019] 检索配置档案 与 [TERM-WS-020] 智能体定制包 依赖。 | G5 |
| R10 | 组合可运行性校验与阻止下单 | 工作流方案的智能员工组合变更必须进行可运行性校验（步骤依赖、必选能力等）；对不可运行方案明确原因并阻止下单与分发。 | G5 |
| R11 | 组合降级提示与实时定价 | 调整智能员工组合时必须提供降级提示（对交付范围、质量、时长的影响），并实时更新价格（按所选智能员工单价组合）。 | G5 |
| R12 | 标准化连接原则 | 系统必须提供一套标准化的外部连接协议/接口规范（即 [TERM-MP-001] 标准连接端点），确保外部工具和数据源只需适配一次即可被所有 [TERM-IA-007] 智能员工 复用；禁止为单一 SaaS 开发无法复用的硬编码连接器。 | G0-2 |

## 5. 用户旅程（Key Flows）

> 只描述关键步骤，避免把 UI 细节散落在这里（UI 细节放附录 A）。
> 硬约束：每个旅程必须标注关联的需求条目编号（MP-0XX）。

### 5.1 旅程 1：管理员浏览与选择工作流方案

- 触发：管理员进入市场页
- 用户目标：浏览全量能力 → 进入工作流详情 → 选择/移除智能员工形成方案并看到价格
- 主流程：
  1) 进入市场页，左侧分类导航，右侧卡片流。（规则：R2）
  2) 卡片展示关键决策要素与成本标签，并提供进入详情入口。（规则：R2）
  3) 进入工作流详情，在方案配置区域选择/移除智能员工，系统实时校验可运行性。（规则：R10）
  4) 系统在组合变更时实时更新价格，并提示降级影响（范围/质量/时长）。（规则：R11）
- 分支/约束：
  - 若组合不可运行，系统必须明确原因并阻止继续订阅/购买。（规则：R10）
- 失败与恢复：
  - 若校验失败或价格计算异常，系统提示并要求恢复到上一个可运行方案。（规则：R10、R11）
- 关联需求：MP-001、MP-010、MP-011

### 5.2 旅程 2：管理员订阅/购买并分配

- 触发：管理员在工作流详情页对可运行方案点击订阅/购买
- 用户目标：订单确认成功后立即完成分配，使能力可被目标对象使用
- 主流程：
  1) 点击订阅/购买并完成订单确认。（需求：MP-002）
  2) 订单确认成功后自动弹出“资源分配”抽屉。（需求：MP-002）
  3) 选择子账号/业务组或员工并确认，系统立即生效并扣减可用额度。（规则：R3）
- 分支/约束：
  - 分配交互保持极简（无搜索）。（需求：MP-002）
  - 若找不到目标，提供跳转去创建新部门/员工。（需求：MP-002）
- 失败与恢复：
  - 订单确认成功但分配失败时，应允许管理员重试分配，且不会重复扣减。（规则：R3）
- 关联需求：MP-002

### 5.3 旅程 3：管理员回收、自动回收与订阅管理

- 触发：员工离职/转岗/资源优化，或团队扩张需要增购/续费
- 用户目标：回收权限、释放额度；在订阅管理中完成增购/续费并查看订单
- 主流程：
  1) 在已分配列表中对条目点击回收。（规则：R6）
  2) 系统立即撤销权限并回流可用额度。（规则：R6）
  3) 可配置“30 天未活跃自动回收”。（规则：R6）
  4) 进入订阅管理入口（如“我的订阅”）完成增购/续费与订单查询。（规则：R4）
  5) 若存在新版本可用，仅提示升级；管理员手动触发后才生效。（规则：R5）
- 分支/约束：
  - 自动回收的“未活跃”口径与触发策略以开放问题结论为准。（见 9 章 Q-MP-001）
- 失败与恢复：
  - 回收/增购/续费失败时，应提示失败原因并允许重试；[TERM-IA-019]可用额度不得出现负数。（规则：R6）
- 关联需求：MP-003、MP-004、MP-005

### 5.4 旅程 4：员工在“我的应用”开始使用

- 触发：员工进入“我的应用（My Apps）”
- 用户目标：只看到自己可用的能力，一键开始使用
- 主流程：
  1) 系统仅展示分配给当前员工的可用能力列表。（规则：R1）
  2) 员工点击卡片进入 Workspace 并开启新[TERM-WS-013]工作会话。（规则：R1）
  3) 空状态：未分配时提示联系管理员分配。（规则：R1）
- 关联需求：MP-008

### 5.5 旅程 5：快速试用与权限声明

- 触发：管理员对未订阅能力选择试用或订阅
- 用户目标：先试用、再订阅；订阅前看清权限边界并确认
- 主流程：
  1) 未订阅卡片展示免费试用入口；点击后立即获得[TERM-IA-023]试用额度并生效。（规则：R8）
  2) 试用到期自动回收。（规则：R8）
  3) 点击订阅/购买前弹出权限声明清单，管理员确认后继续。（规则：R7）
  4) 运行期任何越权操作默认拦截并可追溯。（规则：R7）
- 关联需求：MP-006、MP-007

### 5.6 旅程 6：配置外部营销工具连接

- 触发：市场部负责人希望让智能员工能够直接操作公司的企业微信客户池，或者从巨量引擎后台拉取最新的广告线索。
- 用户目标：打通私域与公域数据，让 Agent 具备实操能力（如自动加好友、发素材），而不仅仅是做文案。
- 主流程：
  1) 管理员进入“市场设置 > 资源连接”页面。
  2) 点击“添加连接”，选择“标准连接器 (MCP)”。
  3) 输入外部系统的[TERM-MP-001]标准连接端点。
  4) 系统自动进行握手测试，成功后用中文展示该服务提供的具体能力列表。
  5) 管理员确认无误后，将该连接绑定给“私域运营助理”工作流。
- 关联需求：MP-012

### 5.7 旅程 7：查看混合计费账单与结算

- 触发：月底，管理员核对市场部的AI预算支出
- 用户目标：清晰区分“固定雇佣成本（人头费）”和“弹性绩效支出（提成）”。
- 主流程：
  1) 管理员进入“订单与发票 > 月度账单”。
  2) 系统展示本月总费用，并提供双轨明细视图：
    - [TERM-MP-006]按岗雇佣费：展示所有订阅制工作流的月租总和。
    - [TERM-MP-005]按结果付费：展示所有结果制工作流的成功交付次数与费用。
  3）点击“结果明细”，可查看每一笔扣费对应的[TERM-MP-002]结果验证器校验凭证。 
- 关联需求：MP-013

### 5.8 旅程 8：引入外部平台智能体

- 触发：运营团队已经在“扣子 (Coze)”或“阿里百炼”上搭建好了一个“小红书爆文助手”，希望引入到公司统一工作台给全员使用。
- 用户目标：低成本引入生态中现成的轻量级工具，无需内部开发。
- 主流程：
  1) 管理员在“扣子 (Coze)”平台发布 Agent 并获取应用连接地址（API URL）和鉴权令牌（Token）。
  2) 回到本系统“市场 > 外部应用接入”，新建一个[TERM-MP-003]外部智能体桥接。
  3) 填入地址和令牌，并配置在工作台的显示名称（如“小红书文案专家”）。
  4) 分配给“新媒体运营组”员工。
  5) 员工在工作台发起对话，系统将消息无缝转发给 Coze 并渲染返回的流式结果。
- 关联需求：MP-014

#### MP-001：市场浏览与详情

- 一句话描述：管理员浏览 [TERM-IA-014] 工作流市场，查看方案详情、薪酬模式与权限声明。
- 关联旅程：5.1
- 主要交互：
  - 市场首页：按业务域（营销/产研/行政）展示卡片；卡片需标记“官方原生”或“外部桥接”（来自 Coze/飞书）。
  - 详情页定价展示：必须明确标识该 Agent 的雇佣模式。
    - 示例A：[TERM-MP-006] 按岗雇佣：30元/月。
    - 示例B：[TERM-MP-005] 按结果付费：50元/有效线索。
- 权限/边界要点：
  - 仅[TERM-IA-001]主账号管理员可见全量市场并进行订阅操作；普通员工不可见市场入口。（规则：R2）
  - 定价展示必须符合混合定价模型定义，明确区分基础费和结果费。（规则：R13）
- 验收标准：
  - AC-MP-001-1：Given 管理员进入市场页，Then 必须出现按业务领域展示卡片，并在卡片上展示详细信息。
  - AC-MP-001-2：Given 管理员进入详情页，When 选择不同的配置套餐，Then 页面必须实时展示该套餐对应的费用构成。
  - AC-MP-001-3：Given 普通员工账号登录，When 尝试访问市场 URL，Then 必须被系统拦截并提示无权限访问。

#### MP-002：订阅下单与权限确认

- 一句话描述：管理员下单订阅，系统生成[TERM-IA-016]订阅实例，且必须经过权限与计费确认。
- 关联旅程：5.2
- 主要交互：
  - 前置权限确认：点击“雇佣/订阅”后，强制弹窗展示[TERM-IA-024]权限声明清单。用户必须勾选“同意授权”方可继续。
  - 预算确认：若包含“按结果付费”项，需在下单页提示设置月度预算上限。
  - 订单确认：订单确认成功后生成订阅实例，状态为“生效中”。
- 权限/边界要点：
  - 订阅前必须向用户展示订阅将获取的权限清单，并支持用户确认后再订阅。（规则：R7）
  - 每次订阅必须生成独立的订阅实例，用于隔离授权、计费与审计。（规则：R7）
  - 计费模式需符合数字化雇佣体系（规则：R13）
- 验收标准：
  - AC-MP-002-1：Given 用户位于下单确认页，When 尚未勾选“同意权限清单”，Then 下单按钮必须处于禁用状态且不可点击。
  - AC-MP-002-2：Given 订单确认成功，When 返回市场，Then 自动弹出[TERM-IA-017]授权分发抽屉，引导管理员立即分配。
  - AC-MP-002-3：Given 选择对象并确认分配，Then 立即生效且[TERM-IA-019]可用额度扣减正确。

#### MP-003：版本策略

- 一句话描述：[TERM-IA-016]订阅实例在使用期保持版本稳定；仅在管理员手动触发升级后生效。
- 关联旅程：5.3
- 主要交互：
  - 在订阅管理中查看当前版本与可升级状态
  - 手动触发升级
- 权限/边界要点：
  - 安装即锁定、被动更新、手动升级。（规则：R5）
- 验收标准：
  - AC-MP-003-1：Given 市场侧版本更新，When 企业[TERM-IA-016]订阅实例存在，Then 不应自动覆盖实例版本。（规则：R5）

- 验收标准：
  - AC-MP-008-1：Given 员工进入“我的应用”，Then 仅展示其已分配能力；点击可进入 Workspace 新[TERM-WS-013]工作会话。（规则：R1）
  - AC-MP-008-2：Given 无分配能力，Then 展示规定空状态文案。（规则：R1）


#### MP-009：多智能员工分步协作与声明（I/O + 信息需求）

- 一句话描述：支持把多个智能员工按步骤串联，系统能识别每步的输入/输出与信息需求，以便稳定编排与校验。
- 关联旅程：5.1
- 主要交互：
  - 上架时声明 I/O Schema
  - 若步骤依赖企业知识：同时声明信息需求与其绑定的检索配置档案/定制包依赖
  - 系统据此完成步骤交接、检索路由与可运行性校验
- 权限/边界要点：
  - 上架能力必须声明 Input/Output Schema；知识依赖必须显式声明信息需求与依赖。（规则：R9）
- 验收标准：
  - AC-MP-009-1：Given 一个上架能力，Then [TERM-WS-014]上架清单（manifest.json）必须存在并声明 [TERM-WS-015]输入输出声明。（规则：R9）
  - AC-MP-009-2：Given 上架包包含企业知识依赖步骤，Then manifest 必须声明 [TERM-WS-018] 信息需求，并引用对应的 [TERM-WS-019] 检索配置档案/可选的 [TERM-WS-020] 智能体定制包。（规则：R9）

#### MP-010：工作流方案配置与可运行性校验

- 一句话描述：管理员可在工作流详情中选择/移除智能员工形成工作流方案，系统实时校验方案是否可运行。
- 关联旅程：5.1
- 主要交互：
  - 方案配置区：选择/移除智能员工
  - 可运行性校验结果：可运行/不可运行 + 原因
- 权限/边界要点：
  - 组合变更必须进行可运行性校验；不可运行必须说明原因并阻止下单与分发。（规则：R10）
- 验收标准：
  - AC-MP-010-1：Given 管理员在工作流详情调整智能员工组合，Then 系统实时返回可运行性校验结果。（规则：R10）
  - AC-MP-010-2：Given 组合不可运行，Then 系统必须明确原因并禁用订阅/购买与分发入口。（规则：R10）

#### MP-011：降级提示与实时定价

- 一句话描述：管理员调整智能员工组合时，系统实时更新价格，并提示对交付范围、质量与时长的影响。
- 关联旅程：5.1
- 主要交互：
  - 价格实时更新
  - 降级影响提示
- 权限/边界要点：
  - 组合变更必须给出降级提示，并实时更新价格（按所选智能员工单价组合）。（规则：R11）
- 验收标准：
  - AC-MP-011-1：Given 管理员调整智能员工组合，Then 系统应实时更新该方案的价格展示。（规则：R11）
  - AC-MP-011-2：Given 管理员移除或替换智能员工，Then 系统必须提示对交付范围/质量/时长的影响。（规则：R11）

#### MP-012：外部资源连接配置(Standardized Connection)

- 一句话描述：管理员通过配置标准接口，将外部营销业务系统（如企业微信、抖音巨量引擎、微信公众号）接入系统，赋予智能员工直接查询线索、管理私域客户等实际工作能力。
- 关联旅程：5.6
- 主要交互：
  - 添加连接：支持输入标准服务地址（Endpoint）。场景示例：输入“企业微信营销助手 MCP 服务”的地址。
  - 能力发现：系统自动解析元数据，列出业务友好的工具列表。示例：自动识别出 search_wecom_customers（查询企微客户）、get_douyin_leads（获取抖音留资线索）、push_article（推送公众号文章）等工具。
  - 绑定：将连接绑定给“市场部”或特定的“私域运营助理”工作流。
- 权限/边界要点：
  - 连接器必须遵循标准化协议（如 MCP），禁止非标硬编码。（规则：R12）
- 验收标准：
  - AC-MP-012-1： Given 一个标准的中国市场营销工具连接地址（如企微/抖音适配器），When 管理员输入并保存，Then 系统能成功握手并准确识别出“查询客户”、“获取线索”等具体工具。

#### MP-013：混合计费模型与结算（Hybrid Billing Engine）
- 一句话描述：支持[TERM-MP-004]数字化雇佣体系下多模式的计量与自动结算。
- 关联旅程：5.7
- 主要交互：
  - 模式 A 结算：按周期（月/年）扣除固定订阅费。
  - 模式 B 结算：任务执行结束后，系统自动调用[TERM-MP-002]结果验证器。
    - 若验证结果=Pass：按单价扣费。
    - 若验证结果=Fail: 不扣结果费。 
  - 账单展示：清晰分列“固定雇佣支出”与“弹性结果支出”。
- 权限/边界要点：
  - 计费逻辑必须严格区分按岗与按结果模式。（规则：R13）
- 验收标准：
  - AC-MP-013-1：Given 结果付费任务执行失败（验证器返回 Fail），When 系统结算费用，Then 严禁产生“结果费”扣款。

#### MP-014：外部智能体桥接（External Agent Bridge）
- 一句话描述： 支持将企业在外部平台（如 Coze/扣子、阿里百炼）已经搭建好的智能体快速引入工作台，使其作为“编外员工”直接参与内部协作。
- 关联旅程：5.8
- 主要交互：
  - 引入智能体：管理员在设置页填入外部平台的应用后台地址（URL）及授权密钥（Token/Key）。
  - 对话体验：员工在工作台像与同事对话一样向其发送指令，系统自动转发并返回图文结果。
  - 安全管控：该“编外员工”的所有回复内容和敏感操作行为（如发送链接），均会被系统自动记录到[TERM-IA-013]系统调用审计日志中，供管理员随时回溯。
- 权限/边界要点：
  - 外部智能体的运行逻辑仍在原平台，但其身份鉴权与行为审计必须受本系统托管。（规则：R14）
- 验收标准：
  - AC-MP-014-1： Given 已配置好一个“扣子(Coze)营销助手”，When 员工在工作台发起对话，Then 能正常获得响应且支持流式输出，同时审计日志能清晰记录“谁（员工）在何时调用了该助手”。

## 8. 风险、依赖与权衡（Risks & Trade-offs）

- 风险 1：分配/回收/自动回收涉及额度一致性与并发。
  - 影响：高
  - 缓解：分配/回收幂等；额度校验；审计链路。（规则：R6）
- 风险 2：[TERM-IA-024]权限声明清单与运行期实际行为不一致会造成安全风险。
  - 影响：高
  - 缓解：运行期默认拦截 + 强校验；必要的声明校验与回归测试。（规则：R7）
- 权衡点：
  - 分配交互“无搜索”提升极简，但在大组织场景可用性下降；本期保持原约束。（需求：MP-002）

### 10.2 [TERM-WS-005]服务端驱动界面（SDUI）与[TERM-WS-016]组件白名单（示意）

- 描述（What）：
  - 服务端驱动界面在下发“可渲染的结构化结果”时，必须携带以下信息类型（不限定字段命名）：

    2) 组件类型：前端据此选择对应的展示组件。

  - 前端只允许渲染“已审核通过”的组件集合；未在集合内的组件一律不渲染。
