## 1. 一句话摘要
为企业提供统一账号与权限体系：以 [TERM-IA-002] 子账号为权限与资产边界，支持员工在多个子账号间切换，并让工作对话等资产在企业内按规则合规留存与交接。

## 2. 目标与指标（Goals & Metrics）
### 2.1 产品目标（Goals）
- G0-1：安全基线：保证企业隔离成立；登录/会话/密码的最小安全要求明确且可执行（规则：R1、R11、R16）。
- G0-2：可追溯基线：关键高风险动作与关键治理动作可被记录、可查询、可用于排障与追责（规则：R14）。
- G0-3：合规基线：确保资产归属权（企业所有）明确，且员工对“资产归属与离职处置规则”充分知情（规则：R10）。
- G0-4：机器身份安全：确立 [TERM-IA-007] 智能员工 作为独立数字实体的 [TERM-IA-028] 代理身份，防止代理劫持与幽灵操作；实施零信任访问控制，确保机器行为可控、可审计（规则：R20）。
- G1：企业可在管理后台开通/续费订阅实例并管理授权额度，并确保购买生效、到期/失败与权益限制口径清晰一致（规则：R4、R17、R18、R19）。
- G2：以子账号作为权限与资产归属主要载体（规则：R2、R3、R5），实现成本边界与访问边界可控。
- G3：资产连续性：构建“私有-公开-归档”的资产流转体系，确保业务知识不随人员流动而断层或流失（规则：R7、R8）。
- G4：通过智能员工职级（L1/L2/L3）约束高风险操作并定义人类确认关口（规则：R6），高风险动作必须留痕可追溯（规则：R14）。
- G5：提供原生闭环的组织管理后台：不依赖第三方系统即可完成子账号/成员/绑定/权限配置与基础审计（规则：R2、R4、R14）。
### 2.2 成功标准（体验结果）
- 成功标准S1：企业主账号可闭环完成“子账号创建、成员入职绑定、权限/资产可见性规则生效”（规则：R2、R3、R5、R7、R8）。
- 成功标准S2：员工可账号密码登录，并在多子账号场景下顺畅切换进入对应工作环境（规则：R9、R11、R16）。
- 成功标准S3：对话资产在隐私分级规则下可控共享，并在员工与子账号解绑触发后可按规则交接给子账号管理员（如配置继任者则继任者同样只读可见）（规则：R8、R10）。
- 成功标准S4：智能员工职级约束对高风险操作（规则：R14）必须可被明确验证（规则：R6）：
  - L1：任何高风险操作（规则：R14）均被拒绝，并产生留痕记录；
  - L2：高风险操作（规则：R14）必须在“人类确认”后才可生效，并产生对应留痕记录；
  - L3：仅在静态白名单条件满足时才可生效；否则拒绝并留痕。
### 2.3 指标定义与统计口径（Metrics，统计口径不强制事件名）
#### 北极星指标（North Star Metric）
- 指标名：组织初始化完成的企业数
  - 对应成功标准：S1
  - 一句话定义：统计周期内，完成“创建≥1子账号 AND 创建并绑定≥1成员 AND ≥1成员完成首次登录”的企业数量（企业去重）。

  - 指标类型：count
  - 分析对象：org（企业/主账号）
  - 时间粒度：day
  - 时间窗：统计周期（如自然周/自然月；按报表选择）
  - 过滤条件：无
  - 维度（可选）：渠道 / 版本 / 行业 / 套餐（TBD）
  - 数据来源：组织与成员相关关键行为事件（见 10.6：组织管理、登录相关）
  - 计算口径：

### 3.2 规范规则表（Normative Rules）
| 规则ID | 规则名称 | 规范描述（必须可验收） | 关联目标（G#，必填）|
|---|---|---|---|
| R1 | 企业隔离 | 不同企业（主账号）之间的成员信息、[TERM-IA-009] 对话资产、文件与数据必须互相不可见（企业隔离/多租户隔离）。 | G0-1 / G2 |
| R2 | 三层账号模型 | 系统必须支持：[TERM-IA-001] 主账号、[TERM-IA-002] 子账号、[TERM-IA-003] 员工账号；员工账号可绑定多个子账号并切换进入对应工作环境。 |G2 / G5 |
| R3 | 资产归属 | 员工在某子账号环境中产生的对话资产归属于该子账号；其可见性与归档按 R7、R8 执行。|G2 / G3 |
| R4 | 授权额度与授权方式 | 主账号拥有订阅实例下的 [TERM-IA-018] 授权额度；系统通过 [TERM-IA-017] 授权分发 将授权额度分配/回收给子账号/员工，并可回溯分发结果。对子账号是否可用某工作流入口仍由 [TERM-IA-006] 授权开关 控制，避免以“手动切分授权额度”作为授权方式。 | G1 / G5 |
| R5 | 入口可见性隔离 | 仅被授权的子账号环境内可见对应工作流入口；未授权则入口不可见。 |G2 |
| R6 | 智能员工职级约束 | L1：只读/建议；L2：产出草稿且必须人类确认后才可执行高风险操作；L3：仅在静态白名单（管理员在设置页面配置）条件下可执行高风险操作；高风险操作定义按 R14 执行。 |G4 |
| R7 | 对话可见性分级 | 对话资产在子账号范围内按三态流转：<br>1) 私有（默认）：仅创建者可见可写；同子账号其他员工不可见；主账号管理员具备“审计只读”可见（只读、不可修改）。<br>2) 团队公开（主动）：创建者可写；同子账号内所有员工只读可见；主账号管理员“审计只读”可见；跨子账号默认不可见（除非管理员显式授权，口径由相关 PRD/规则定义）。<br>3) 解绑归档（解绑触发）：该子账号下创建者的私有对话转为只读归档；对子账号管理员只读可见（如配置继任者则继任者同样只读可见）；创建者在被解绑后不可见。<br>注：对子账号管理员的删除/隐藏权限按管理后台能力定义，不改变上述默认可见性口径。 | G3 |
| R8 | 解绑归档触发与范围 | 当员工与某子账号解绑触发（含离职/禁用/转岗导致解绑）时：<br>1) 归档与交接：该员工在该子账号下产生的所有“私有”对话自动转为只读归档，并交接给该子账号管理员只读可见；若管理员为该员工指定了“继任者”，则继任者同样获得只读可见权限。<br>2) 禁用触发范围：禁用员工账号时，系统应视为该员工与其绑定的所有子账号均发生解绑，并分别执行对应子账号范围内的归档与交接。<br>3) 团队公开处理：其发布的“团队公开”对话保持公开状态不变（作为团队公共知识留存），但作者标识变更为“已离职成员”；其管理权（删除/隐藏）移交给子账号管理员。<br>4) 原子性：若归档/交接流程因数据量/系统异常导致超时或失败，系统必须拒绝完成禁用（保持账号为启用态）并提示失败原因；管理员可重试归档/禁用操作，且需保证幂等。<br>5) 误禁用恢复（资产返还）：管理员对已禁用账号执行“启用”时，系统必须提供“是否返还历史资产”的可选项（默认不返还）。若选择返还，则对本次禁用触发的归档资产执行回滚：将仍处于“解绑归档”状态且未被删除/隐藏的对话恢复为该员工“私有”且可读写，并撤销因解绑归档触发而授予子账号管理员/继任者的只读可见权限；已被管理员删除/隐藏的对话保持其最终状态，不再返还。 | G3 / G0-3 |
| R9 | 登录与多子账号选择 | 员工账号密码登录：若仅绑定 1 个子账号，直接进入工作台；若绑定多个子账号，默认自动进入“上次活跃”的子账号环境（跳过选择页），仅在无历史记录时展示选择页；工作台内必须提供切换子账号入口。 |G2 |
| R10 | 隐私合规提示 | 首次登录或用户协议摘要中必须明确告知：工作对话为企业资产；员工与子账号解绑触发（含离职/禁用/转岗导致解绑）后按规则留存与交接。 |G0-3 / G3 |
| R11 | 会话与暴力破解防护 | 7 天滑动过期（活跃不需重登）；30 天强制重新验证，重新验证=重新输入密码；同账号 10 分钟内连续输错 5 次则锁定 15 分钟并展示倒计时。 |G0-1 |

| R13 | 删除子账号约束 | 子账号仅当“无成员且无资产”时允许删除；否则必须禁止并提示原因。 |G2 / G5 |

| R19 | 权益限制与超额处理 | 当有效 [TERM-IA-018] 授权额度 不足以覆盖已分配额度（含到期/降配/退款等情况）时，系统进入“超额”状态：禁止新增分发；管理员必须通过“续费补足”或“回收分发”解除超额；进入限制模式后仅允许管理员访问组织设置与计费页进行续费/补缴，其它业务入口按限制策略执行。 | G1 |
| R20 | 代理身份与零信任 |  智能员工必须拥有独立的 [TERM-IA-028] 代理身份（基于加密技术的机器身份），不得仅依赖人类账号的凭证（User Impersonation）执行跨系统操作；所有 Agent 对外部系统（如数据库、SaaS API）的访问必须遵循 [TERM-IA-029] 零信任访问 原则：默认拒绝、最小权限、动态获取短期令牌，并留下独立的审计轨迹（Agent ID + Transaction ID）。 |G0-4/G0-2|

| R22 | 检索与证据的权限一致性 | 当工作流在运行时检索企业知识源并返回引用证据时：系统**必须**确保“能看到证据的人”具备对应来源的访问权限；不得在无权限情况下返回可推断来源内容的引用摘要。 | G0-1 / G4 |
| R23 | 智能体定制包的权限与变更留痕 | [TERM-WS-020] 智能体定制包的创建/发布/升级/回滚必须受权限控制（管理员或被授权角色），且每次变更必须记录变更人、时间、版本号与变更说明；被工作流引用的版本不得被静默替换。 | G4 / G0-3 |
| R24 | 管理员自我保护 | 系统必须禁止当前操作者对自身 [TERM-IA-003] 员工账号 执行“禁用/删除”等会导致其失去登录与管理能力的操作；若操作者需要离职或停止担任管理员，必须先将主账号管理员权限移交给另一名处于启用状态的员工账号后方可执行。系统必须保证每个 [TERM-IA-001] 主账号 始终至少存在 1 个处于启用状态的主账号管理员：当任一操作将导致“启用的主账号管理员数=0”时，系统必须拒绝该操作并提示原因与处理步骤（先完成权限移交/恢复其他管理员账号后再操作）。 | G0-1 / G5 |
#### R14-Fields：高风险动作留痕最小字段（MVP）
- 执行记录 ID（用于定位一次动作的审计记录）
- 智能员工标识（Agent ID，用于唯一定位一个智能员工实例）
- 事务标识（Transaction ID，用于关联一次对外访问/一次高风险动作的链路）
- 智能员工名称（Agent Name）
- 动作类型（Action Type）：例如“写入文件”“发起网络请求”“修改系统配置”（示例即可，不要求用户理解底层实现）
- 动作对象（Target）：例如“文件路径 / 目标域名 / 资源标识”
- 结果（Result）：Allow（通过） / Deny（拒绝） / Modify（人工修改） / Fail（执行失败） / Timeout（确认超时）
- 发生时间（Timestamp）
- 触发者（Actor）：员工 / 智能员工 / 系统
- 归属子账号（Sub-account）
- 归属工作流（Workflow）
- 任务标识 / 对话资产标识（用于关联一次任务链路）

## 5. 用户旅程（Key Flows）
### 5.1 旅程 1：员工登录与多环境智能路由
- 触发：员工访问登录页。
- 用户目标：以最快路径进入目标工作环境（熟客自动直达，新客便捷选择），避免无意义的交互。
- 主流程：
  1) 用户输入账号/密码登录（系统校验密码要求，规则：R11、R16）。
  2) 系统根据账号绑定情况执行智能跳转（规则：R9）：
     - 单环境用户： 若员工仅绑定 1 个 [TERM-IA-002] 子账号，直接进入工作台。
     - 多环境用户（熟客）： 若员工绑定多个子账号且系统检测到“上次活跃记录”，跳过选择页，直接进入该子账号环境，并以全局提示（Toast）告知：“已自动进入 [市场部] 环境”。
     - 多环境用户（首次/无记录）： 若无历史记录，跳转至“选择子账号”卡片页，用户点击卡片后进入。
  3）用户在工作台主动发起环境切换操作，从列表中选择目标子账号，系统即时刷新至新环境。
- 分支/约束：
  - 若识别为首次登录或管理员重置过的临时密码，在步骤 2 之前强制拦截，跳转至修改密码页（规则：R12）。
  - 服务受限拦截： 若企业处于限制模式（规则：R19），普通员工登录后拦截进入工作台，并展示全局提示页：“企业服务受限，请联系管理员处理。”
- 失败与恢复：
  - 登录失败： 若登录失败（如密码错误），提示错误原因并展示剩余重试次数（规则：R11）。
  - 改密失败： 提示失败原因并允许重试；未完成改密不得进入工作台（规则：R12、R16）。
  - 忘记密码： 若用户点击“忘记密码”，提示“如忘记密码，请联系您的企业管理员重置。”（规则：R16）。
  - 环境异常： 若自动跳转的目标子账号已被禁用或删除，系统自动回退至“选择子账号”页面。  
- 关联需求：IA-005
### 5.2 旅程 2：管理员极速组织初始化（子账号/成员）
- 触发：管理员需要快速批量建立组织架构
- 用户目标：在3分钟内完成50人以上的部门创建与账号下发。
- 主流程：
  1) 管理员进入“子账号管理”，点击新建并输入名称（如“华东销售部”），创建业务单元。
  2) 管理员进入“成员管理”，选择 “批量导入” 标签页。
  3) 数据准备与输入：
     - 场景 A（有数据）： 直接从现有电子表格（如 Excel/WPS）中复制“姓名”与“登录账号（邮箱/手机）”列。
     - 场景 B（无数据）： 点击“下载标准模板”，在本地填充后复制内容。
     - 场景 C（手动）： 按“姓名 + 分隔符 + 登录账号（邮箱/手机）”逐行录入（分隔符支持空格或逗号）。
  4) 将数据粘贴至文本框，系统自动解析并生成预览表格；管理员在底部的“所属子账号”下拉框中勾选“华东销售部”（规则：R2）。
  5) 点击确认导入。系统自动创建对应 [TERM-IA-003] 员工账号，并按登录账号类型发送包含临时密码的入职邀请（邮箱→邮件；手机→短信）（规则：R12）。
- 分支/约束：
  - 若粘贴内容格式有误（如邮箱格式错误），系统在预览表格中高亮报错行，允许管理员直接在表格内修正，无需退回原文件修改。
- 失败与恢复：
  - 部分导入失败： 若解析成功但部分数据入库失败（如邮箱已被占用），系统仅提交成功条目，并生成“失败名单”供管理员下载或在界面上修正后重试。
  - 邮件发送失败： 若账号创建成功但邀请邮件发送超时或被拦截，允许管理员在成员列表中点击“重新发送邀请”。 
- 关联需求：IA-001、IA-007、IA-008、IA-004
### 5.3 旅程 3：资产可见性与离职交接
- 触发：员工主动共享知识，或员工离职导致账号被禁用。
- 用户目标：确保高价值信息在团队内沉淀，确保离职后企业资产不流失。
- 主流程：
  1) 主动共享：员工在工作台中将某个 [TERM-IA-010] 私有 状态的对话切换为 [TERM-IA-011] 团队公开；此时，同子账号的其他成员可只读查看此内容。（规则：R7）
  2) 离职触发：管理员在后台“成员管理”中对某员工执行“禁用”操作。
  3) 交接确认：系统弹出确认框，提示将触发 [TERM-IA-012] 解绑归档，并需确认或指定一位“资产继任者”。（规则：R8）
  4) 归档生效：确认后，该员工在对应子账号下的所有“私有”对话自动转为只读归档状态，并立即对管理员可见。（规则：R8）
- 分支/约束：
  - 无资产场景：若该员工名下无私有对话数据，系统自动跳过资产交接确认步骤，直接执行账号禁用。
  - 自身保护：账号无法禁用自身；若需离职，必须先将管理员权限移交至其他成员。（规则：R24）
- 失败与恢复：
  - 归档失败：若因数据量过大导致资产归档超时或失败，系统将回滚禁用操作并提示错误，避免出现“账号已封禁但资产悬空”的中间状态。
  - 误操作/重新启用：若需恢复账号（含误操作场景），管理员点击“启用”时，系统在弹窗中提供“是否同时返还历史资产”的可选项（默认不勾选）；若确认返还，则仅返还仍处于“解绑归档”状态且未被删除/隐藏的对话：恢复为该员工“私有”且可读写，并撤销因解绑归档触发而授予管理员/继任者的只读可见权限；已被管理员删除/隐藏的对话保持最终状态不返还。（规则：R8）
- 关联需求：IA-004
### 5.4 旅程 4：主账号管理员开通/续费订阅实例（商业化）
- 触发：管理员在添加成员/发起授权分发时发现有效 [TERM-IA-018] 授权额度不足，或需主动扩充团队规模。
- 用户目标：顺畅完成订阅实例的开通/续费，使授权额度尽快生效，确保业务不中断。
- 主流程：
  1) 管理员查看当前 [TERM-IA-016] 订阅实例 状态、有效 [TERM-IA-018] 授权额度 与已分配额度（含超额提示）。
  2) 点击“开通/续费订阅实例”入口，选择方案/周期与目标额度（系统展示预计生效时间与账单预览）。
  3) 确认订单详情（计费周期、应付金额/账单摘要、权益范围）并提交开通/续费请求。
  4) 系统生成订单记录与账单记录；订阅实例状态进入“待生效”。
  5) 订阅实例生效后，系统即时更新有效授权额度；管理员返回“成员管理/授权管理”继续未完成的邀请或分发。
- 分支/约束：
  - 权限控制：仅主账号管理员或拥有“订阅管理”权限的角色可见入口。
  - 限制模式拦截：若企业处于限制模式（到期且宽限期结束），仅允许管理员进入计费页完成续费/补缴后恢复（规则：R19）。
  - 超额状态：若已分配额度 > 有效授权额度，禁止新增授权分发；需先续费补足或回收分发解除超额（规则：R19）。
- 失败与恢复：
  - 开通/续费失败：请求失败或超时，订单状态为“失败/可重试”，不改变有效授权额度；管理员可在订单列表重试，无需重新配置。
  - 权益延迟：若订单已成功但订阅实例未及时生效（异步处理/网络延迟），界面提供“刷新状态”并触发后台对账/重试；状态最终以订阅实例为准。
  - 幂等：重复提交同一开通/续费请求必须幂等（同一 `Idempotency-Key` 返回同一订单与结果）。
- 关联需求：IA-010
### 5.5 旅程 5：智能员工的高风险授权
- 触发：员工向 L2 级别的 [TERM-IA-007] 智能员工 下达高风险指令（如“批量发送邮件”）。
- 用户目标：既要利用 AI 提效，又要通过人工确认防止灾难性错误。
- 主流程：
  1) 员工在对话框输入高风险指令。
  2) 系统识别该指令涉及高风险操作，且该 Agent 职级为 L2。（规则：R14、R6）
  3) Agent 暂停执行，生成操作草稿（如邮件预览），并在对话流中返回一个“待确认”卡片。
  4) 员工点击卡片查看详情，确认无误后点击“批准”。
  5) Agent 执行发送动作；系统在后台记录 [TERM-IA-013] 系统调用审计（包含操作人、Agent、时间、批准结果=Allow）。（规则：R14）
- 分支/约束：
  - 权限不足（L1）：若 Agent 职级为 L1，系统直接拒绝执行并提示权限不足，同时记录审计日志（结果=Deny）。（规则：R6、R14）
  - 人工干预：若员工点击“拒绝”或“修改”，Agent 终止当前执行，系统记录审计日志（结果=Deny/Modify）。
- 失败与恢复：
  - 执行失败：若经批准后 Agent 执行动作失败（如 API 超时），在对话流中提示错误，并提供“重试”按钮，无需重新生成草稿；系统记录审计日志（结果=Fail）。
  - 状态失效：若草稿生成后长时间（如 24 小时）未操作，卡片状态变更为“已失效”，需重新发起指令；系统记录审计日志（结果=Timeout）。
- 关联需求：IA-003

- 主要交互：
  - 身份发放：系统在创建智能员工实例时，自动生成一对加密密钥（公私钥）；私钥必须存储在受保护的密钥存储设施中且不可导出（具体实现由架构文档定义）。
  - 鉴权流程：当 Agent 需要访问外部资源（如数据库、第三方 API）时，不直接使用员工的 Session Token；而是使用其私钥签名的请求向“身份服务/鉴权服务（由架构文档定义）”换取短期访问令牌（Short-lived Access Token）。
  - 审计记录：系统日志必须明确区分指令发起人（操作员工）与实际执行主体（智能员工），以确保追责链路清晰。
- 权限/边界要点：
  - 本条需求受规则R20约束。（规则：R20）
  - 严禁 Agent 长期持有静态凭证（API Key）；必须通过身份服务/鉴权服务（由架构文档定义）动态获取短期访问令牌。
- 验收标准：
  - AC-IA-011-1：Given 智能员工需要访问受保护资源，When 发起请求，Then 系统必须验证其加密签名的机器身份，而非仅验证当前操作员工的身份。
  - AC-IA-011-2：Given 发生一次 Agent 操作，When 查看审计日志，Then 必须能看到独立的 Agent ID，且能追溯到该操作是由哪位员工授权/触发的。
  - AC-IA-011-3：Given Agent 疑似被劫持或异常，When 管理员吊销该 Agent 身份证书，Then 该 Agent 对所有资源的访问权限应立即失效（无需修改员工密码）。
### 6.13 IA-012：企业知识源访问控制（[TERM-WS-017]）
- 一句话描述：企业知识源必须可被授权与审计，且归属/可见性边界与子账号一致。
- 关联规则：R21、R22
- 验收标准：
  - AC-IA-012-1：Given 一个企业知识源归属到子账号 A，When 子账号 B 的员工尝试引用该来源，Then 默认不可见且不可访问，除非管理员显式授权。
  - AC-IA-012-2：Given 无权限员工触发检索，Then 系统不得返回可推断敏感内容的引用摘要，并记录拦截审计。

### 6.14 IA-013：智能体定制包治理（[TERM-WS-020]）
- 一句话描述：定制包必须版本化、可控发布、可回滚，且变更留痕。
- 关联规则：R23
- 验收标准：
  - AC-IA-013-1：Given 一个定制包被工作流引用，When 定制包升级，Then 必须记录变更说明并保留可回滚到上一版本的能力。
  - AC-IA-013-2：Given 非授权角色尝试发布/升级定制包，Then 系统必须拒绝并记录审计。

## 8. 风险、依赖与权衡（Risks & Trade-offs）
- 风险 1：对话可见性与解绑归档/资产交接需要多处能力保持一致（入口展示、资产归属、权限校验与审计记录）。
  - 影响：高
  - 缓解：以规则 R7、R8 为核心规则来源，建立规则校验 + 关键链路联动 + 全流程演练的验收覆盖。
- 风险 2：[TERM-IA-013] 系统调用审计 依赖系统具备稳定记录与查询能力，否则无法满足企业追溯诉求。
  - 影响：中-高
  - 缓解：先落地 MVP 最小字段闭环，后续升级到步骤级留痕（vNext）（规则：R14）。
- 权衡：
  - 默认私有提升员工安全感，但可能降低团队沉淀效率；通过“主动公开 + 解绑归档”平衡。
  - L2 人工确认降低自动化效率，但能显著降低高风险操作，带来的损失与合规风险（规则：R14）。

### 10.3 IA-020：组织记忆管理（归属层级/权限/删除与留痕）
- 描述：
  - 记忆归属层级采用三档：对话资产（[TERM-IA-009]）级 / 子账号（[TERM-IA-002]）级 / 主账号（[TERM-IA-001]）级（术语与定义统一见 SSOT 或「组织记忆管理」独立 PRD；本处仅做引用）。
  - 每条记忆最小信息：记忆编号、类型（知识/经验/流程）、来源（员工输入/智能员工总结/工具结果）、创建人、创建时间。
  - 权限规则：管理员可按角色（管理员/员工）与职级（L1/L2/L3）配置各层“可读/可写/可删除”。
  - 留痕要求：对记忆读取/写入/删除记录“谁在何时对哪条记忆做了什么、结果如何”。
### 10.4 IA-021：单次任务内容上限（配额与超限策略）
- 描述：
  - 为一次任务设置“可提交给 AI 的内容上限”（token budget）：要求可计算、可限制、可提示，不要求用户理解 token 细节。
  - 上限建议拆分约束：用户输入、系统附加记忆/知识、工具返回结果。
  - 超限时至少支持一种策略：压缩（总结）、截断、拒绝并引导。
  - 记录内容负载指标（Context Load）：衡量是否接近上限。
### 10.5 IA-022：关键动作留痕字段升级（步骤级留痕）
- 背景：MVP [TERM-IA-013] 系统调用审计只能粗粒度记录高风险动作；为支撑排障与分层查看，需要升级为任务内步骤级留痕。
- 新增记录信息（在 MVP 基础上补充）：
  - 任务链路编号；步骤编号与父步骤编号；工具信息（工具名/动作类型）；输入/输出摘要（不可逆摘要）；策略裁决（策略编号、允许/拒绝、原因）；执行者信息（员工/智能员工/系统、所属子账号与主账号）。
