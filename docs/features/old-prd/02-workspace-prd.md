# **产品需求文档: 智能工作台 (Intelligent Workspace)**

状态: Frozen (V9.5) | 优先级: P0 | 关联模块: 统一身份与通行证

版本说明: V9.5 Final - 确立“服务端驱动 (Server-Driven)”的交互原则；明确追问建议与图表渲染均由 Agent 工作流动态决策，移除前端硬逻辑；整合 UI 视觉规范。

## **1. 核心目标 (Core Objective)**

构建 AI Native 的协同执行环境。

核心诉求是让用户通过自然语言和文件轻松下达指令，并能通过可视化的进度展示和自适应的结果呈现，获得透明、高效、可信的工作体验。同时，提供 IDE 级别的布局管理和多业务线导航能力。

## **2. 功能需求 (Functional Requirements)**

## **2.0 核心隐喻：工作区即操作系统 (Workspace as OS) [NEW]**

* **Session → Process (进程)**:
  * 用户与 Agent 的交互不再是简单的“对话”，而是启动一个或多个**智能进程**。
  * 进程具备生命周期：`Created` -> `Running` (后台执行) -> `Suspended` (挂起/等待人工) -> `Completed`。
  * **持久化**: 所有进程状态必须持久化存储，支持断点续传（类似 `kubectl exec` 连接到运行中的 Pod）。
* **Context → File System (虚拟文件系统)**:
  * Agent 的记忆不应只依赖 Token 上下文，必须拥有对 **工作区文件系统 (VFS)** 的读写权限。
  * Agent 是“员工”，有权直接修改项目文件（Write/Patch），而不仅仅是提出建议。

### **2.1 超级输入框 (Super Composer)**

**目标**: 统一的多模态指令入口，集成工作流选择与文件管理。

* **F2.1.1 独立附件栏 (Attachment Area)**:
  * **位置**: 位于文本输入框的**正上方**（当有文件被选中时出现）。
  * **交互**: 上传的文件以**缩略图卡片**横向排列，不与文本混排。包含类型图标、文件名及删除按钮。
* **F2.1.2 多模态输入体验 (Multimodal Input)**:
  * **A. 拖拽上传**: 全屏响应，文件拖入浏览器任意区域，显示“释放以添加”遮罩。
  * **B. 按钮上传**: 点击工具栏 [➕] 按钮选择文件。
  * **C. 剪贴板粘贴 (Clipboard Paste)**:
    * **支持内容**: 用户在操作系统中复制截图、文件或文本后，在输入框聚焦状态下按 Ctrl+V (Cmd+V)。
    * **系统行为**: 自动识别剪贴板内容类型，将其转换为附件卡片展示在附件栏。
  * **限制规则**:
    * **容量限制**: **单文件最大 512 MB** (Benchmark: ChatGPT/Microsoft 365)。
    * **数量限制**: **单次对话最多 10 个文件**。超限时 Toast 阻断提示。
* **F2.1.3 工作流选择器 (Workflow Selector) - Simplified**:
  * **布局结构**: **垂直分层布局 (Vertical Layout)**。
    * **上层**: 多行文本输入区域。
    * **下层 (Bottom Toolbar)**: 工具栏，包含附件与选择器。
  * **具体位置**: **底部工具栏的最左侧**。
  * **视觉流向**:
    ```text
    ...................................... 
    ........... 文本输入区域 ............... 
    ......................................
    ------------------------------------------
    [➕ 附件] [🤖 合同审查 ⌄]               [➤ 发送]
    ```
  * **形态**: **透明背景的胶囊按钮 (Ghost Pill)**。
    * **默认状态**: 显示 [图标] 当前工作流名称 + 下拉箭头 ⌄。
    * **视觉反馈**: 鼠标悬停 (Hover) 时显示浅色背景。
  * **交互逻辑 (Interaction)**:
    * **点击**: 弹出一个**简单的下拉列表 (Dropdown Menu)**。
    * **内容**: 仅展示用户已订阅的工作流列表（通常 < 10 个）。
    * **操作**: 直接点击列表项即可切换。
  * **切换行为 (Behavior)**:
    * **场景 A (零态/新对话)**: 立即更新当前 Agent 设定，UI 无跳转。
    * **场景 B (对话进行中)**: 选中新工作流后，自动**开启一个新会话**，避免上下文污染。
* **F2.1.4 混合输入能力**:
  * 支持长文本编辑 (Shift+Enter 换行) 和 Markdown 语法。
* **F2.1.5 生成控制 (Generation Control)**:
  * **发送后**: 发送按钮变为 **“停止 (■)”** 图标。
  * **点击停止**:
    * 若在生成中：立即断开连接，保留已生成内容。
    * 若在等待中：取消请求，**自动恢复**输入框内的草稿。

### **2.2 自适应流式反馈 (Adaptive Response)**

**目标**: 根据 Agent 的决策智能切换展示形态，实现“生成式 UI (Generative UI)”。

* **F2.2.1 文本流**: 打字机特效，首字延迟 < 1s。
* **F2.2.2 生成式界面 (Generative UI) - Server-Driven [UPDATE]**:
  * **核心定义**: Agent 输出的不是静态 Markdown，而是动态的 **UI 描述 JSON** (Server-Driven UI)。
  * **A. 动态微件 (Dynamic Widgets)**:
    * 场景: 当用户问“销售数据如何？”
    * 行为: 渲染一个可交互的 `<DashboardCard type="sales_overview" data={...} />`，支持筛选和钻取，而非静态表格。
  * **B. 交互式表单 (Interactive Forms)**:
    * 场景: Agent 需要用户确认复杂信息（如审批差旅费）。
    * 行为: 渲染一个 `<ApprovalForm fields={[...]} />`，包含 [✅ 批准] [❌ 驳回] 按钮，用户操作直接回调给 Agent 进程。
* **F2.2.3 智能编辑器 (Smart Editor) - Creative/Biz Edition**:
  * **核心定义**: 一个与聊天上下文解耦、支持富文本编辑的持久化工作区。
  * **A. 触发机制 (Agent-Driven Trigger)**:
    * **判定源头**: 编辑器的开启权限完全由 **智能体 (Agent)** 根据当前任务的性质决定。
    * **UI 表现**: 仅当智能体判定当前内容为“交付物”时，在消息卡片底部显示显眼的操作按钮。
      * **按钮样式**: [📝 在编辑器中打开] (Open in Editor)。
    * **用户动作**: **仅当用户点击该按钮时**，右侧面板才会展开。
  * **B. 交互模式 (Interaction Modes)**:
    * **编辑态 (Edit Mode)**: 用户点击按钮后，右侧分屏打开，内容自动“提拔”进入编辑器。
      * **纯净体验 (Creative Focus)**:
        * **视觉风格**: 模拟 A4 纸或白板的**无干扰书写区**。
        * **悬浮工具栏**: 仅在用户**选中文字**时浮出（加粗/斜体/标题/列表/引用）。
      * **交付工具栏 (Export Actions)**:
        * **位置**: 编辑器顶部右侧常驻。
        * **功能**:
          * [📋 复制格式化内容] (Copy as HTML/Markdown)
          * [📥 导出] (下拉菜单: PDF, Word, Markdown)
      * **锁定逻辑**: 用户键入时锁定 Agent 写入；超时 60秒或断网时自动释放锁。
      * **\[NEW\] 行内意图 (Inline Intent / Co-authoring)**:  
        * **交互**: 用户在编辑器中**选中一段文字**时，自动浮出 AI 指令气泡（类似 Notion AI / Google Docs Comment）。  
        * **功能**: 提供快捷指令 \[✨ 润色\] \[📏 缩短\] \[🌐 翻译\] 或输入自定义 Prompt（如"语气更委婉一点"）。  
        * **局部更新**: Agent 仅流式刷新**被选中的段落**，而不是全篇重写，保持上下文连贯性。
  * **C. 版本与差异 (Version & Diff)**:
    * **快照机制**: 每次 Agent 或用户完成一次编辑，系统自动保存一个版本快照 (Snapshot)。
    * **更新反馈**: 当用户在左侧 Chat 要求“修改一下语气”时，Agent 更新编辑器内容后，编辑器顶部短暂显示 ✅ 已更新 Toast。
    * **差异高亮**: 在“修订模式”下，使用**绿色背景**标记新增，**红色删除线**标记移除。
  * **D. 上下文管理 (Context Hygiene)**:
    * **Rule**: Agent 在生成回答时，仅读取编辑器的**当前最新版本** (Current State)。
* **F2.2.4 评价与反馈 (Feedback Loop)**:
  * **粒度**: **逐条消息 (Per-Message)**。
  * **UI 位置**: 消息卡片底部工具栏 (Action Bar)，与 [复制] 并列。
  * **交互逻辑**:
    * **👍 点赞**: 点击即高亮。后端记录该条问答对为“高价值资产”，用于“最佳实践榜”排名。
    * **👎 点踩**: 点击后图标变红，可选弹出轻量气泡 [不准确] [不安全] [格式错误]，供模型调优。
* **F2.2.5 引用与溯源 (Citations)**:
  * **显示规则**: 当 Agent 输出内容基于上传文件或联网搜索时，必须在文中包含 [1] [2] 引用角标。
  * **交互**:
    * **Hover**: 显示来源摘要（文件名/网页标题）。
    * **Click**: 滚动到附件区高亮对应文件，或打开来源网页。
* **F2.2.6 智能追问 (Smart Follow-up) - Agent 驱动**:
  * **触发机制**: 依赖后端流式响应中的 `suggestion_chips` 事件。若 Agent 判定无后续建议（为空），前端不展示任何 UI。
  * **显示时机**: 仅在 AI 消息输出结束 (`final: true`) 后显示。
  * **形态**: 消息卡片下方独立的“胶囊按钮”行。
  * **交互**: 点击胶囊，直接将内容填入输入框并自动发送。
* **F2.2.7 人机协同中断 (Request for Help / Syscall) [NEW]**:
  * **定义**: 当 Agent 遇到权限不足、信息缺失或置信度低时，**主动挂起进程 (Suspend)** 并请求介入。
  * **UI 表现**:
    * 在消息流中弹出一个 **"需要协助 (Assistance Required)"** 的卡片。
    * **明确阻碍**: 例如 “无法访问 `/finance/2025_salary.csv`，权限被拒绝”。
    * **操作按钮**: [✅ 授权访问] [❌ 拒绝并终止] [✍️ 手动输入数据]。

### **2.3 推理透明化 (Thought Transparency / CoT)**

**目标**: 建立 B2B 场景下的“白盒信任”。Agent 不应是黑盒，必须展示“思考路径”。

* **UI 形态**: 消息卡片顶部默认展示 **“思考过程 (Reasoning Steps)”** 折叠面板。  
* **交互逻辑**:  
  * **进行中**: 实时流式展示当前步骤（如 "🔍 正在查阅《员工手册》2024版..." \-\> "⚖️ 正在比对竞业协议条款..."）。  
  * **已完成**: 自动折叠为摘要 "✅ 已完成 4 步推理"，点击可展开查看完整日志。  
  * **错误处理**: 若某步骤失败，自动展开并高亮卡点，允许用户手动干预（"点击此处授权访问云盘"）。

### **2.4 全局导航与侧边栏 (Sidebar & Navigation)**

**目标**: 提供完整的应用导航和工作区管理能力。

* **F2.4.1 侧边栏布局架构**:
  * **宽度**: 固定宽度 (260px)。
  * **区域划分**:
    1. **顶部固定区 (Fixed Top)**:
       * [+] 新建对话 (主按钮)。
       * [Toggle] 临时对话 (无痕模式)。
       * **系统导航菜单 (Nav Menu)**:
         * [🏪 工作流市场] (Marketplace)
         * [📊 数据洞察] (Insights)
         * **[⚙️ 组织设置] (Settings)**: **[UPDATE]** 原“三方连接”入口升级。点击进入身份管理后台（包含成员、部门、连接）。
    2. **中部滚动区 (Scrollable Middle)**:
       * 历史会话列表 (按时间分组)。
    3. **底部固定区 (Fixed Bottom)**:
       * 身份卡片 (当前子账号信息)。
       * 个人设置菜单 (点击卡片弹出)。
  * **F2.4.1.1 响应式折叠 (Responsive Collapse)**:
    * **规则**: 当右侧 **智能编辑器 (Smart Editor)** 展开时，侧边栏应**自动收起**（或切换为仅图标模式 Compact Mode），以优先保证中间对话流和右侧编辑器的屏幕空间。
    * **恢复**: 当编辑器关闭时，侧边栏自动恢复展开状态。
* **F2.4.2 主题切换 (Theme Toggle)**:
  * **位置**: 移入底部 **身份卡片** 的弹出菜单中。
  * **功能**: 点击在 **☀ 浅色** / **🌙 深色** / **💻 跟随系统** 之间切换。
* **F2.4.3 会话列表管理 (Session Management)**:
  * **Hover 操作**: 鼠标悬停在会话项上时，显示 [•••] 更多菜单。
  * **菜单功能**:
    * **✏️ 重命名**: 允许用户自定义会话标题（默认由 Agent 自动生成摘要）。
    * **🗑️ 删除**: 软删除会话（进入回收站或直接隐藏），列表不再显示。
    * **📌 置顶 (Pin)**: [可选 MVP] 将高频会话固定在列表顶部。
* **F2.4.4 历史消息加载策略 (History Loading)**:
  * **分页机制**: 进入会话时，仅预加载最近20条消息。  
  * **滚动加载**: 用户向上滚动触顶时，触发无感加载 (Infinite Scroll) 历史记录。
* **F2.4.5 任务管理器 (Process Dashboard) [NEW]**:
  * **入口**: 侧边栏顶部常驻 "任务管理器 (Activity Monitor)" 图标。
  * **面板视图**:
    * **运行中 (Running)**: 展示后台 Agent 状态（e.g., "竞品监控 Agent - 运行中 3h 12m"）。
    * **挂起 (Suspended)**: 等待用户确认的任务。
  * **控制能力**: 支持用户对进程进行 [⏸️ 暂停] [▶️ 恢复] [⏹️ 强制终止 (Kill)] 操作。

### **2.5 发起新对话 (Start New Chat)**

* **零态视图**:
  * 清空主界面，展示品牌 Logo。
  * **动态快捷指令**: 基于 **F2.1.3** 选定的工作流，展示 3-4 个特定的 Prompt 建议。
* **懒创建**: 仅在用户发送第一条消息时才在数据库创建 Session ID。

### **2.6 隐私与共享 (Privacy)**

* **F2.6.1 隐私默认值 (Privacy by Default)**:
  * **默认状态**: 所有新会话默认均为 **🔒 私有 (Private)**。
  * **可见范围**: 仅当前用户（及租户管理员审计）可见。
* **F2.6.2 团队分享 (Team Sharing)**:
  * **入口**: 顶部 Header 右上角提供 [🌐 分享] 按钮 (Share Button)。
  * **操作**: 用户点击后，弹出确认框 "是否将此对话对团队成员公开？"。
  * **状态变更**: 确认后，图标变为 [🌐 团队公开]，此时同子账号/业务组的同事可查看（只读）。
* **F2.6.3 临时对话模式 (Temporary/Incognito Mode)**:
  * **触发**: 通过侧边栏 **F2.4.1** 的开关开启。
  * **视觉反馈**:
    * 顶部 Header 变为 **深灰色/特殊色**，明确显示 "🚫 临时模式 - 内容不保存"。
    * 输入框显示水印 "当前对话历史将在刷新后消失"。
  * **数据逻辑**:
    * **不入库**: 消息仅在内存中流转，**不写入** PostgreSQL 历史记录表。
    * **无历史**: 刷新页面或点击“新建对话”后，内容彻底销毁，不可恢复。
    * **功能限制**: 禁用“团队公开”分享功能。

## **3. 通用界面交互规范 (UI/UX Guidelines)**

*本章节提取自原交互设计规范，作为全局开发基准。*

* **G1. 布局尺寸约束 (Layout Constraints)**:
  * **侧边栏**: 展开状态宽度固定 **240px**；收起（图标模式）宽度固定 **60px**。
  * **顶部栏 (Header)**: 高度固定 **64px**。

* **G2. 视觉状态反馈 (Visual States)**:
  * **悬停机制 (Hover Logic)**:
    * 无论主题切换开关位于何处，系统内所有可点击元件（按钮、链接、列表项）需遵循统一反馈：
    * **☀ 浅色模式**: 悬停时背景色 **变暗一级** (Darken)。
    * **🌙 深色模式**: 悬停时背景色 **变亮一级** (Lighten)。
    * **光标**: 所有交互元素悬停时，鼠标必须变为手型 (`cursor: pointer`)。

* **G3. 消息体渲染增强 (Message Rendering)**:
  * **流式光标**: AI 生成过程中，文本末尾必须跟随闪烁的光标符 (▍)，直到 `final: true`。
  * **代码块交互**:
    * 必须支持语法高亮。
    * 右上角常驻 **[复制]** 按钮，点击后显示 "✅ 已复制" Tooltip 反馈。

## **4. 异常体验规范**

* **网络中断**: 输入框上方红条提示，自动重连。
  * **重连中 (Reconnecting)**: 顶部显示黄色提示条 "连接中断，正在尝试重连..."，此时禁止用户发送新消息，但允许浏览历史内容。
  * **重连失败**：超过 30秒未恢复，提示条转红 "连接失败，请刷新页面"，并提供 [刷新] 按钮。
* **生成失败**: 消息底部显示 Regenerate 按钮。
* **文件失败**: 附件卡片变红，Tooltip 显示原因。

## **5. 数据埋点需求**

* chat_new: 创建新会话 (参数: workflow_id)。
* chat_temp_mode: 开启/关闭临时模式。
* chat_share: 用户将私有会话转为公开。
* msg_like / msg_dislike: 消息点赞/点踩 (参数: msg_id, workflow_id)。
* file_upload: 上传文件 (参数: count, file_type, size, source='paste'|'drag'|'click')。
* editor_open: 用户点击按钮打开智能编辑器 (原 canvas_open)。
* editor_edit: 用户手动编辑内容。
* editor_export: 用户点击导出/复制。
* workflow_switch: 切换工作流 (参数: from_id, to_id)。
* sidebar_nav_click: 点击系统导航 (marketplace/insights/settings)。
* theme_change: 主题切换。
