# **产品需求文档: 工作流市场 (Workflow Marketplace)**

状态: Frozen (V6.0) | 优先级: P1 | 关联模块: 身份模块, 智能工作台, 数据洞察

版本说明: V6.0 Asset Lifecycle \- 补全“手动回收”与“增购续费”功能，形成完整的 B2B 资产管理闭环；分配交互保持极简（无搜索）。

## **1\. 核心目标 (Core Objective)**

构建一个分级可视、闭环驱动的 AI 能力分发中心。

核心价值在于解决 B2B 场景下的\*\*“能力分发”与“资产管理”\*\*问题。  
初期版本采用 “管理员主导 (Admin-Driven)” 模式：管理员统一采购、分配及回收席位，员工直接使用已分配的资源。

## **2\. 双重视图需求 (Dual View Requirements)**

### **2.1 主账号视图 (Admin View) \- "采购与管理"**

**目标**: 提供全局的工作流浏览、订阅、分配及全生命周期管理能力。

* **F3.1.1 全量市场浏览**:  
  * **核心布局**: 左侧为业务职能分类导航（法务、产研、营销、HR），右侧为卡片流。  
  * **卡片信息**: 仅展示核心决策要素。  
    * **基础信息**: 图标、名称、简介、供应商。  
    * **成本标签**: \[¥ 299/席位/月\] (强调席位成本，非 Token)。  
* **F3.1.2 订阅与分配 (Subscription & Assignment)**:  
  * **订阅动作**: 点击 \[购买席位\] \-\> 选择数量 \-\> 支付。  
  * **分配动作 (License Allocation)**:  
    * 支付成功后，自动弹出“资源分配”抽屉。  
    * **A. 分配流程**:  
      1. **选择对象**: 下拉选择**已存在**的 **子账号/业务组** (如 "法务组") 或 **员工**。  
      2. **确认分配**: 系统扣减可用席位库存，目标对象即刻获得权限。  
    * **B. 身份配置引导**:  
      * 若下拉列表为空或找不到目标，提供跳转链接 \[+ 去创建新部门/员工\] 至 **\[01-身份管理\]**。
    * **C. 版本快照策略 (Versioning Strategy) [Technical Constraint]**:  
      * **安装即锁定 (Snapshot on Install)**:企业订阅某个 Agent 时，系统应锁定该 Agent 当前的 Prompt/Workflow 版本快照。
      * **被动更新**:市场中原 Agent 的更新不应自动覆盖企业已分配的实例，除非管理员在“订阅管理”中手动点击 [升级版本]。
  * **回收动作 (License Revocation) \[NEW\]**:  
    * **场景**: 员工离职、转岗或资源优化。  
    * **交互**: 在“已分配列表”中，点击某条目右侧的 \[回收\] 按钮。  
    * **结果**: 立即剥夺该对象的权限，席位释放回“可用库存池”，可重新分配给他人。  
  * **自动回收**: 可设置“若 30 天未活跃，自动回收席位”。  
* **F3.1.3 订阅管理 (Subscription Management) \[NEW\]**:  
  * **入口**: 市场页顶部的 \[我的订阅\] 标签。  
  * **功能列表**:  
    * **增购席位 (Add Seats)**: 随着团队扩张，为已有 Agent 购买更多 License。  
    * **续费 (Renew)**: 延长订阅有效期。  
    * **发票与订单**: 查看历史采购记录和下载发票。
* **F3.1.4 快速试用 (Quick Trial) [NEW]**:
  * **策略**: 基于运营配置的固定策略（如：所有 Agent 默认提供 7 天全功能试用）。
  * **交互**: 在未订阅 Agent 的卡片上显示 **[⚡ 免费试用]** 按钮。
  * **行为**: 点击后立即生效，自动在“资源分配”中获得一个“试用期 License”，过期后自动回收。
* **F3.1.5 权限声明清单 (Permissions Manifest) [NEW]**:
  * **交互**: 用户点击 [订阅] 时，系统弹出“权限请求确认框”，列出该 Agent 申请的系统能力。
  * **权限类型**:
    * 🌐 **Networking**: 允许访问互联网 (白名单域名)。
    * 📂 **File System**: 允许读写 `/project/docs` 目录。
    * 🧠 **Memory**: 允许读取团队长期记忆库。
    * 🤖 **Inter-Agent**: 允许调用其他 Agent (如 "Writer Agent" 调用 "Search Agent")。
  * **安全约束**: 运行时 (Runtime) 必须实施沙箱隔离，默认拒绝未声明的系统调用。

### **2.2 员工视图 (Employee View) \- "我的应用"**

**目标**: 纯粹的工具启动器 (Launchpad)，仅展示可用工具，无干扰。

* **F3.2.1 "我的应用" (My Apps)**:  
  * **展示逻辑**: 仅展示已分配给当前员工的所有可用 Agent。  
  * **列表样式**: 网格卡片布局，清晰展示图标和名称（类似 Okta/企业微信工作台）。  
  * **操作**: 点击卡片直接跳转至 **智能工作台 (Workspace)** 开启新会话。  
  * **空状态**: 若未分配任何 Agent，显示提示 *"暂无可用工具，请联系企业管理员分配"*。

## **3. Agent 通信协议 (Inter-Agent Communication)**

**目标**: 支持 Agent 之间的管道式协作 (Unix Pipe Philosophy)。

* **F3.3.1 管道机制 (Piping)**:
  * 允许将一个 Agent 的输出 (Output) 直接作为另一个 Agent 的输入 (Input)。
  * **场景**: `Research Agent` (输出 JSON 报告) -> `Report Writer Agent` (生成 PDF)。
* **F3.3.2 接口声明**:
  * 所有上架 Agent 必须在 `manifest.json` 中声明其 **Input Schema** 和 **Output Schema**，以便系统自动编排。

## **4\. 数据埋点 (Analytics)**

* market\_view: 进入市场页 (Admin only)。  
* license\_assign: 管理员分配席位。  
* license\_revoke: 管理员手动回收席位。  
* subscription\_add: 增购席位。  
* purchase\_order: 管理员下单采购。
